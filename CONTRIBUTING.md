# Contributing to Notiway

Thank you for your interest in contributing to Notiway! This guide will help you get started.

## Ways to Contribute

### Report Bugs
Found something broken? [Open a bug report](https://github.com/Notiway/Community/issues/new?template=bug_report.md) with steps to reproduce.

### Request Features
Have an idea? [Submit a feature request](https://github.com/Notiway/Community/issues/new?template=feature_request.md) and describe your use case.

### Contribute to Plugins
All Notiway plugins are open source. You can:
- Fix bugs in existing plugins
- Improve documentation and examples
- Create custom plugins

Browse available plugin repositories at [github.com/orgs/Notiway/repositories](https://github.com/orgs/Notiway/repositories).

### Improve Documentation
Documentation improvements are always welcome. The documentation site source is available at [Notiway/Notiway.FE.Portal](https://github.com/Notiway/Notiway.FE.Portal).

## Plugin Development

### Getting Started
1. Fork the plugin repository you want to contribute to
2. Clone your fork locally
3. Create a branch for your changes
4. Make your changes and test them
5. Submit a pull request

### Guidelines
- Follow existing code style and conventions
- Write clear commit messages
- Include tests where applicable
- Update documentation if your change affects behavior

## Questions?

Join our [Discussions](https://github.com/Notiway/Community/discussions) to ask questions or connect with the community.
